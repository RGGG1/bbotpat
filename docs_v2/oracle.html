1. Overview

The HiveAI Oracle exposes a small, opinionated set of on-chain & market signals:

  • HMI – Hive Mind Index (modernised fear & greed indicator).
  • Dominance – BTC dominance versus individual tokens or token baskets.
Spot + derivatives aware Low-frequency friendly (12h cadence) Designed for dashboards & bots

2. Base URL

Replace <BASE_URL> with your HiveAI node address, for example: 

http://<BASE_URL>:8000

All endpoints below are relative to that base URL.

If you run behind a reverse proxy (e.g. Nginx on https://hiveai.yourdomain.com), you can call the API directly on that host instead of :8000.

3. HMI endpoints

Latest HMI value
GET
/api/hmi/latest

Example:

curl "<BASE_URL>:8000/api/hmi/latest"

Response:

{
  "timestamp": "2025-11-22T05:15:07Z",
  "hmi": 59.8,
  "band": "stable"
}
HMI history
GET
/api/hmi/history?days=N

Example (last 30 days):

curl "<BASE_URL>:8000/api/hmi/history?days=30"

4. Dominance (DOM) endpoints

Token universe & macros
GET
/api/dom/options

Example:

curl "<BASE_URL>:8000/api/dom/options"
Latest dominance for BTC vs single token
GET
/api/dom/latest?pair=BTC_SOL

Example:

curl "<BASE_URL>:8000/api/dom/latest?pair=BTC_SOL"
Latest dominance for custom baskets
GET
/api/dom/latest?x=BTC&y=ALTS
/api/dom/latest?x=BTC&y=SOL,UNI
Dominance history
GET
/api/dom/history?pair=BTC_SOL&days=N
/api/dom/history?x=BTC&y=ALTS&days=N

Historical ranges (min / max) for BTC vs single tokens and BTC vs ALTS are already available and power the dominance UI on the main page.

5. Update frequency & polling

Signals are recomputed every 12 hours, aligned with the main HiveAI dashboard updates (07:15 and 19:15 CEST).

  • Polling more often than every few minutes is usually unnecessary.
  • Reasonable polling: once every 5–15 minutes.
  • Historical queries can be cached client-side between runs.

6. Custom dominance pairs

By default, full historical ranges are maintained for core pairs (BTC vs individual tokens, BTC vs ALTS).

If you need long-term dominance history for a specific pair or pair combo (e.g. SOL vs ETH+BNB), it is available on request. We will:

  • Begin tracking that pair/basket combination.
  • Expose a dedicated oracle endpoint for it.

To request new pairs, reach out via the same channel where you obtained this node, or open an issue on the repository.
